開場：你要先記住的一句話

一句話總結： Serialization 就是把 Python 物件（model/data/scaler）存成檔案；用 dill 你可以把「模型＋必要資料」打包成 bundle，之後載回來立刻評估/推論。

你要背的 3 個重點：

  1. 盡量存 bundle（model + test set + feature_names + scaler + notes）
  2. 版本要記：python/sklearn/numpy（不然很容易 load 爆）
  3. 不要 load 來路不明的 dill（安全性）

關鍵字（Keywords，後面一小句就懂）

  • serialization / deserialization：把 Python 物件「存成檔」/「從檔案還原」
  • dill：比 pickle 更能存各種 Python 物件的序列化工具
  • bundle：把「model + 必要資料/設定」打包成一包，方便重現結果
  • version compatibility：不同 python/sklearn/numpy 版本，載入可能直接爆
  • security：載入 dill = 可能執行任意程式碼；不明來源不要開

import dill

存：dump bundle（推薦）

bundle = {
  "model": model,
  "data": {"test_data": X_test, "test_labels": y_test},
  "feature_names": list(X_test.columns) if hasattr(X_test,'columns') else None,
  "scaler": scaler if 'scaler' in globals() else None,
  "notes": {"comment": "wk2 module2c"}
}

with open('bundle.dill','wb') as f:
    dill.dump(bundle, f)

載：load bundle

with open('bundle.dill','rb') as f:
    b = dill.load(f)

model = b['model']
X_test = b['data']['test_data']
y_test = b['data']['test_labels']

常見踩雷（Common pitfalls）

  1. 版本不相容：sklearn/numpy 版本不同 → load 可能報錯
  2. 檔案太大：不要把整個 raw dataset 都塞進去
  3. 安全性：不要 load 不明來源 dill（可能執行惡意 code）

小練習（Quick check｜含簡單解法 code）

1) 你能存一個 model 並載回來 type() 還是一樣嗎？

import dill

# 假設你已經有 model
with open('model_only.dill', 'wb') as f:
    dill.dump(model, f)

with open('model_only.dill', 'rb') as f:
    model2 = dill.load(f)

print('before:', type(model))
print('after :', type(model2))
print('same type?', type(model) == type(model2))

2) 你能從 bundle 裡拿出 X_test，跑一次 model.predict(X_test) 嗎？

import dill

with open('bundle.dill','rb') as f:
    b = dill.load(f)

model2 = b['model']
X_test2 = b['data']['test_data']

pred = model2.predict(X_test2)
print(pred[:10])
print('n_preds:', len(pred))
